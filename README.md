# SUMMER-OF-CODE-PROJECT-25B3945
JUST A SUMMER PROJECT 
Week 1 was mostly just setup. I installed Python 3.11 and VS Code, went through the first few CS50P lectures to get the basics down, and then started with the OpenCV tutorial from freeCodeCamp. The first thing I actually built was just opening the webcam and displaying the feed in a window. I also messed around with drawing stuff on the frames like rectangles and circles and tried getting the FPS to show up. Honestly this week was more reading and watching than actual coding but it gave me a decent base.
Week 2 is where it got more interesting. I set up MediaPipe and tried to get hand detection running. The thing that tripped me up the most was that nothing was getting detected at all even though the webcam was clearly working. Spent a while debugging and turns out the issue was that OpenCV reads frames in BGR format but MediaPipe expects RGB so I was basically feeding it the wrong colors the whole time. Once I added the cvtColor conversion it started working immediately which was both a relief and a little embarrassing. By the end of the week I had the skeleton showing up on my hand in the live feed which felt pretty satisfying.
Week 3 was the hardest one so far. The goal was to actually extract the 21 landmark coordinates and use them to detect gestures. The confusing part was that the coordinates MediaPipe gives you are not in pixels, they're normalized floats between 0 and 1, so I had to multiply them by the frame width and height to get actual positions I could work with. Another thing that confused me for a bit was the y-axis being flipped, meaning y increases downward in image coordinates, so a finger pointing up actually has a smaller y value at the tip than at the joint. Once I figured that out the finger detection logic started making sense. I got 5 gestures working: fist, open palm, pointing, peace sign, and thumbs up, and they print to the terminal as I make them.
Week 4 the main task was turning all of that into a proper reusable module called gestures.py instead of having everything in one messy script. The big upgrade was fixing the pinch detection because in week 3 I was using raw pixel distances which completely broke whenever I moved my hand closer or further from the camera. The fix was to normalize the distance by dividing it by the size of the hand itself so the threshold stays consistent no matter how far the hand is. I also had to handle left vs right hand separately for the thumb logic since the direction it bends depends on which hand you're using. I split everything into gestures.py for the logic and test_gestures.py to actually test it with the webcam, the idea being that the game code later just imports gestures.py without touching anything else.
